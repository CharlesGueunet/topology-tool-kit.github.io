30 /*
31  * Morse-Smale complex developer quick guide:
32  *
33  * What is the architecture?
34  * -------------------------
35  * The DiscreteGradient class contains the basic structures to define and build
36  * a discrete gradient. It also has several functions that decrease the number
37  * of unpaired cells as parallel post-processing steps. Even more work is done
38  * on the gradient with a sequential simplification step. Finally, it is able
39  * to build critical points and vpaths.
40  * Its files are:
41  * DiscreteGradient.cpp
42  * DiscreteGradient.h
43  *
44  * The AbstractMorseSmaleComplex class contains whatever is common
45  * between the MorseSmaleComplex2D class and the MorseSmaleComplex3D class
46  * e.g. parameters, configuration functions, input and output data pointers.
47  * In particular, it contains a DiscreteGradient attribute and a Triangulation
48  * attribute.
49  * Its files are:
50  * AbstractMorseSmaleComplex.cpp
51  * AbstractMorseSmaleComplex.h
52  *
53  * The MorseSmaleComplex2D class is derived from the AbstractMorseSmaleComplex
54  * class. It is specialized in building the Morse-Smale complex on 2D
55  * triangulations. This class uses the DiscreteGradient attribute to build a
56  * valid discrete gradient before building the MSC outputs itself i.e.
57  * critical points, 1-separatrices, segmentation.
58  * Its files are:
59  * MorseSmaleComplex2D.cpp
60  * MorseSmaleComplex2D.h
61  *
62  * The MorseSmaleComplex3D class is derived from the AbstractMorseSmaleComplex
63  * class. It does the same job as the MorseSmaleComplex2D class but on 3D
64  * triangulations. Note that this class has a function
65  * computePersistencePairs() to get the saddle-saddle pairs of the data.
66  * It adds the saddle-connectors to the 1-separatrices and adds another output
67  * for 2-separatrices.
68  * Its files are:
69  * MorseSmaleComplex3D.cpp
70  * MorseSmaleComplex3D.h
71  *
72  * The MorseSmaleComplex class is derived from the AbstractMorseSmaleComplex
73  * class. It is a convenience class that detects the dimensionality of the data
74  * and uses the correct implementation of the Morse-Smale complex computation
75  * (between MorseSmaleComplex2D and MorseSmaleComplex3D).
76  * Its files are:
77  * MorseSmaleComplex.cpp
78  * MorseSmaleComplex.h
79  *
80  * How to build the gradient?
81  * --------------------------
82  * Everything that concerns the gradient is in the DiscreteGradient class.
83  * In order to build a valid discrete gradient you need to first set the data
84  * pointers to the input scalar field, offset field and triangulation. You need
85  * to set also the data pointers to the output critical points, 1-separatrices,
86  * 2-separatrices and segmentation. Additional parameters can be configured
87  * like an iteration threshold, options to have PL-Compliant extrema and
88  * saddles, an option to enable collecting of persistence pairs or
89  * post-processing of the saddle-connectors. Note that they all have default
90  * values that correspond to a standard scenario. Like any other TTK
91  * module, the level of debug and the number of threads can be adjusted to suit
92  * your needs.
93  * Once all the parameters and data pointers are set, the function
94  * buildGradient() builds the discrete gradient. As a substantial number of
95  * unpaired cells is expected, it is strongly recommended to use after this
96  * function the function buildGradient2() and after that the buildGradient3()
97  * function if the input dataset is in the 3D domain.
98  * Finally, you can apply reverseGradient() to auto-detect the PL critical
99  * points and impose that the gradient is PL-Compliant (except on the
100  * boundary). See the related publication "The Topology ToolKit" for further
101  * details.
102  * Examples of such usage of the DiscreteGradient class can be found in
103  * the execute() function of the MorseSmaleComplex2D class and
104  * MorseSmaleComplex3D class as these classes need to compute a discrete
105  * gradient to build their own outputs.
106  *
107  * Where is the simplification algorithm?
108  * --------------------------------------
109  * The main steps of the gradient simplification algorithm are stored in the
110  * reverseGradient() function in the DiscreteGradient class. More informations
111  * can be found in each simplify-like function as the process is slightly
112  * different depending on the index of the critical points involved:
113  * simplifySaddleMaximumConnections() for reversal of (saddle,...,maximum)
114  * vpaths.
115  * simplifySaddleSaddleConnections1() for reversal of
116  * (2-saddle,...,1-saddle) vpaths.
117  * simplifySaddleSaddleConnections2() for reversal of
118  * (1-saddle,...,2-saddle) vpaths.
119  *
120  * How to add a scalar field on any output geometry?
121  * -------------------------------------------------
122  * First, go to AbstractMorseSmaleComplex.h and add the void* pointer to a STL
123  * container (e.g. vector) as a class attribute. In the code, the attributes
124  * of the same output are grouped together and are prefixed by its name.
125  * So for example, the outputSeparatrices1_points_smoothingMask_ variable
126  * represents the smoothingMask scalar field that is associated to the points
127  * of the 1-separatrices of the Morse-Smale complex. As you just added a new
128  * attribute, you need to update the setter corresponding to the output to
129  * add this new element:
130  * The setters available are:
131  * setOutputCriticalPoints()
132  * setOutputSeparatrices1()
133  * setOutputSeparatrices2()
134  * setOutputMorseComplexes()
135  * As you added a new attribute in the class, you need to give it a default
136  * value in the constructor in AbstractMorseSmaleComplex.cpp (typically nullptr
137  * for a pointer).
138  * Now, you need to overload the same setter than previously but in
139  * MorseSmaleComplex.h this time in order to propagate the new field
140  * information to the concrete implementations. As the MorseSmaleComplex2D and
141  * MorseSmaleComplex3D classes are both derived from AbstractMorseSmaleComplex
142  * they already have the updated version of the function.
143  * Now that you have access to the data pointer inside the actual
144  * implementation you can cast it from void* to dataType* and modify it to
145  * your convenience.
146  *
147  * What part of the code is parallel?
148  * ----------------------------------
149  * From a global point a view, the part of the code that builds the gradient
150  * as well as the two post-processing steps are accelerated by OpenMP (if
151  * available).
152  * The gradient simplification step is mostly sequential except the
153  * initialization of internal structures in the initialize-like functions which
154  * is done is parallel with OpenMP.
155  * Then, each output of the Morse-Smale complex is computed in parallel with
156  * OpenMP except the 2-separatrices of a 3D dataset that require heavy
157  * synchronisation.
158  * Finally, the generation of the geometry for the visualization is done
159  * sequentially.
160  * Complete list:
161  * In the DiscreteGradient class:
162  * buildGradient()
163  * buildGradient2()
164  * buildGradient3()
165  * initializeSaddleMaximumConnections()
166  * initializeSaddleSaddleConnections1()
167  * initializeSaddleSaddleConnections2()
168  * In the MorseSmaleComplex2D class:
169  * getSeparatrices1()
170  * setAscendingSegmentation()
171  * setDescendingSegmentation()
172  * In the MorseSmaleComplex3D class:
173  * getSeparatrices1()
174  * getAscendingSeparatrices2()
175  * getDescendingSeparatrices2()
176  * setAscendingSegmentation()
177  * setDescendingSegmentation()
178  */
